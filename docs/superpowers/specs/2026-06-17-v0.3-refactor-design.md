# NekoECAT Converter v0.3 重构设计规格

## 概述

对 C++ Qt5 前端进行全面重构，修复核心 Bug，优化架构，完善工程化。

## 目标

1. 修复所有导致崩溃和功能失效的 Bug
2. 消除 UI 线程阻塞，实现真正的异步操作
3. 清理死代码，统一架构
4. 添加 Python 后端单元测试
5. 完善工程配置（pyproject.toml）
6. 增强用户体验（实时进度、错误处理）

## 架构设计

### 前端架构（C++ Qt5）

```
gui/
├── src/
│   ├── main.cpp              # 入口，命令行解析
│   ├── mainwindow.h/cpp      # 主窗口（布局+交互）
│   ├── worker.h/cpp          # 异步转换工作线程
│   └── core/
│       └── theme.h/cpp       # 主题样式（纯数据，无 UI 依赖）
├── CMakeLists.txt            # 构建配置
└── build/                    # 构建输出
```

### 关键设计决策

1. **异步导入**：`onImport()` 使用 QProcess + 信号槽，不阻塞 UI
2. **步骤状态管理**：存储 stepNum 为成员变量，步骤切换时更新样式
3. **样式统一**：所有按钮样式通过 `Theme::genButtonStyleSheet()` 应用
4. **错误处理**：Python 进程不存在时显示友好提示
5. **进度协议**：Python 输出 `{"progress": N, "msg": "..."}` 格式

## 修复清单

### Bug 1: connectSignals 未调用
- **位置**：`mainwindow.cpp` 构造函数
- **修复**：在构造函数末尾调用 `connectSignals()`
- **影响**：过滤和搜索功能完全失效

### Bug 2: UI 线程阻塞
- **位置**：`mainwindow.cpp:onImport()`
- **修复**：改为异步 QProcess，使用信号槽通信
- **影响**：导入时界面冻结长达 30 秒

### Bug 3: genButtonStyleSheet 未应用
- **位置**：`mainwindow.cpp:buildBottomBar()`
- **修复**：在生成按钮上应用样式
- **影响**：生成按钮无样式

### Bug 4: stepNum 未存储
- **位置**：`mainwindow.cpp:buildSidebar()`
- **修复**：将 stepNum 存储为成员变量
- **影响**：步骤切换时编号圆圈无 active 样式

### Bug 5: 死代码 pages/
- **位置**：`gui/src/pages/`
- **修复**：删除整个目录
- **影响**：CMakeLists.txt 未包含，代码冗余
